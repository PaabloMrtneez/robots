# Guia Tecnica SwiftLend

- El codigo debe estar escrito en TypeScript.
- El servicio HTTP debe usar Node.js y Express.
- Los nombres de variables, funciones y propiedades JSON deben usar camelCase.
- La arquitectura debe separar dominio, aplicacion, infraestructura e interfaces.
- Todas las entradas externas deben validarse con Zod.
- La logica de negocio debe ser testeable sin levantar el servidor HTTP.
- Los tests deben usar Jest.
